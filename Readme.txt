程式說明:

首先字的部分，每次都是一個已知配一個未知，

出現過的字就不會再出現，除非已經輪完一輪。

兩個的順序我採取隨機的方式呈現，以免使用者刻意只答前面或後面的字。

按下Enter後，格子會顯示顏色，綠色代表答對，紅色代表答錯。

顏色使用了另一個Thread，不然delay的時候會影響到其他動畫。

如果想要replay，只要輸入replay即可。

另外如果是以replay的方式重玩，之前答對過的紀錄會留著。

但如果直接關掉程式，記錄在下次開啟的時候就會覆寫。

這樣一來，只要重複replay然後答對73題，就能把所有的未知字辨識完。

Reassembler的部分，先將known_word.txt讀進來，

再將第一題的輸出讀進來，然後sort即可。

接著就照著座標的順序一直找下去，如果known有該座標就用known，

如果輸出有該座標就用輸出，如果都沒有就是 __ 。


遇到的問題:

一開始不知道要覆寫PaintComponent才能畫圖，

後來發現後，就再開了一個class繼承JPanel，然後覆寫就好了。

然後是讀圖片檔，因為known有51個，unknown有73個，要一個一個讀很麻煩，

後來發現可以用file.list得到資料夾內所有檔案的名稱，就簡單多了。

然後是Reassembler的部分，一開始不知道怎麼讓使用者在cmd輸入要讀的檔案，

後來發現main(Srring[] args)的args就是輸入的字串。
